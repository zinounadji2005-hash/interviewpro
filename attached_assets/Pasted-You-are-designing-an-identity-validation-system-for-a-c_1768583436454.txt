You are designing an identity validation system for a career-interview web platform.

Goal:
Prevent duplicate accounts for the same person while allowing different users with similar or identical names to register fairly.

Core Principle:
Names are NOT unique identifiers. Email and verified identity signals are authoritative. Name matching is used only as a validation signal, not as a hard restriction.

---

1. Account Creation Rules

- Email address is the primary unique identifier.
- Do NOT block account creation based on name alone.
- Allow multiple users with identical or similar names as long as their emails are different and verified.

---

2. CV vs Account Name Validation (AI-Based)

When a user uploads a CV:

- Extract the full name from the CV using AI text analysis.
- Normalize both names:
  - Lowercase
  - Trim whitespace
  - Remove duplicate spaces
  - Handle middle names and initials
- Compare the CV name with the registered account name.
- Generate a Name Match Confidence Score (0–100).

Confidence Score Interpretation:
- 80–100 → Strong match → proceed automatically
- 60–79 → Partial match → show a warning and request user confirmation
- Below 60 → Weak match → block CV processing and request correction

---

3. Decision Logic

- Never automatically accuse the user of fraud.
- Use respectful, professional language in all messages.
- Example warning message:
  “The name on your CV partially matches your account name. Please confirm that this CV belongs to you.”

---

4. Anti-Abuse Safeguards (Non-Intrusive)

Prevent multiple accounts for the same individual by:
- Enforcing unique email addresses
- Optional OAuth (Google / LinkedIn) linking
- Detecting repeated abuse patterns (optional, future)

Do NOT:
- Use name alone as a database unique constraint
- Block common names (e.g., Mohammed Ahmed)
- Expose whether another user exists with the same name

---

5. Data & Architecture

- Store:
  - full_name (original)
  - full_name_normalized
  - cv_name_extracted
  - name_match_score
- Supabase remains the source of truth.
- All validation logic must run server-side.

---

6. UX & Trust

- Keep feedback neutral and professional.
- Focus on data accuracy and user trust.
- Allow future expansion for:
  - Manual review
  - Identity verification
  - Admin override

---

Outcome:
A fair, scalable, and abuse-resistant identity validation system that supports similar names without compromising platform integrity or user experience.
